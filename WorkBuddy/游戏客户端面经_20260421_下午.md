## 📬 上期思考题答案

> 上期问：一款 5v5 MOBA 手游用帧同步开发，观战者加入时比赛已进行了 5 分钟（约 4500 帧）。怎么让观战者快速追上当前进度？直接从第 1 帧快进到第 4500 帧会有什么问题？

**答案**：直接从第 1 帧追到第 4500 帧，理论可行但实际会**卡死好几秒甚至十几秒**——因为你要在一瞬间执行 4500 帧的全部逻辑（移动、技能、碰撞、Buff、AI……），CPU 撑不住。

正确做法是 **关键帧快照 + 增量追帧**，分两步：

**第一步：服务器定时存快照**

服务器每隔 N 帧（比如每 300 帧 ≈ 20 秒）存一份完整的世界状态快照（所有角色的位置、血量、Buff、技能 CD 等）：

```
帧号:    0    300    600    900    ...    4200    4500(当前)
快照:    S0    S1     S2     S3           S14
```

**第二步：观战者从最近的快照恢复，只追剩余帧**

观战者加入时，服务器发给他第 4200 帧的快照 S14。观战者先把世界状态恢复到 S14，再从 4200 帧开始追帧到 4500 帧——只需要追 300 帧，而不是 4500 帧。

```
不用快照：追 4500 帧 → 可能卡 8-10 秒
用快照：  追 300 帧  → 约 0.5 秒搞定
```

**额外优化**：追帧期间跳过所有渲染、特效、音效，只跑纯逻辑，速度还能再快 3-5 倍。

**面试加分**：提到快照的空间开销控制——每个快照可能几十 KB 到几百 KB，用环形缓冲区只保留最近 N 个快照，老的自动覆盖。

## 🎯 面试题（⭐⭐ 中级）

**Q：说说 Lua 的元表（metatable）和 `__index` 元方法。游戏里怎么用元表实现一套"类与继承"系统？如果 `__index` 指向一个函数和指向一个表，有什么区别？**

### 参考答案

Lua 没有 `class` 关键字，但用 **表 + 元表** 就能模拟出完整的面向对象。核心就靠一条规则：

> 当你访问表中不存在的 key 时，Lua 会去查元表的 `__index`。

先看最简单的例子，搞懂 `__index` 的查找链：

```lua
-- 基类 Animal
local Animal = {}
Animal.__index = Animal  -- 关键！让实例能找到类方法

function Animal.New(name)
    local self = setmetatable({}, Animal)
    self.name = name
    return self
end

function Animal:Speak()
    print(self.name .. " says: ...")
end

-- 使用
local cat = Animal.New("Tom")
cat:Speak()  -- Tom says: ...
```

查找过程画个图就清楚了：

```
cat 表本身:  { name = "Tom" }
                ↓ cat:Speak() → cat 表里没有 Speak
                ↓ 查元表的 __index → 指向 Animal 表
                ↓ Animal 表里有 Speak → 找到了！

cat (实例)
  ├── name = "Tom"          ← 实例自己的数据
  └── [metatable] → Animal
        ├── __index = Animal  ← 指向自身
        ├── New()
        └── Speak()          ← 在这里找到
```

再加上**继承**——子类的 `__index` 指向父类：

```lua
-- 子类 Dog 继承 Animal
local Dog = setmetatable({}, { __index = Animal })
Dog.__index = Dog

function Dog.New(name, breed)
    local self = Animal.New(name)       -- 先调父类构造
    setmetatable(self, Dog)             -- 再设成 Dog 的实例
    self.breed = breed
    return self
end

function Dog:Speak()                    -- 重写（多态）
    print(self.name .. " says: Woof!")
end

function Dog:FetchBall()                -- 子类新方法
    print(self.name .. " is fetching!")
end

local d = Dog.New("Buddy", "Golden")
d:Speak()      -- Buddy says: Woof!   （走 Dog 的 Speak）
d:FetchBall()  -- Buddy is fetching!  （走 Dog 的 FetchBall）
```

查找链变成了**三层**：

```
d (实例) → Dog (子类) → Animal (父类)
  │           │             │
  │           │             └── Speak() ← 被覆盖了，不会走到这
  │           ├── Speak()    ← 重写版
  │           └── FetchBall()
  └── name, breed  ← 实例数据
```

**`__index` 指向函数 vs 指向表的区别**：

| 方式 | 写法 | 特点 |
|------|------|------|
| **指向表** | `mt.__index = SomeTable` | 直接去那个表里查 key，简单高效 |
| **指向函数** | `mt.__index = function(t, k) ... end` | 每次缺失都调函数，灵活但有开销 |

函数形式常用于**延迟加载**或**访问控制**：

```lua
-- 用函数做"默认值 + 访问日志"
local defaults = { hp = 100, mp = 50, speed = 10 }

local mt = {
    __index = function(t, k)
        print("[WARN] 访问了未设置的属性: " .. k)
        return defaults[k]  -- 返回默认值
    end
}

local player = setmetatable({ hp = 200 }, mt)
print(player.hp)     -- 200（实例自己有，不走 __index）
print(player.speed)  -- [WARN] 访问了未设置的属性: speed
                     -- 10（走了 __index 函数）
```

**面试加分**：提到 `rawget(t, k)` 可以绕过元表直接读表，`rawset(t, k, v)` 绕过 `__newindex`。面试官会觉得你真的用过。

## 📖 知识讲解：闭包与 upvalue

### 什么是闭包？

一句话：**闭包 = 函数 + 它引用的外部局部变量（upvalue）**。

```lua
function MakeCounter()
    local count = 0          -- 这就是 upvalue

    return function()        -- 这个匿名函数 + count = 闭包
        count = count + 1
        return count
    end
end

local c1 = MakeCounter()
print(c1())  -- 1
print(c1())  -- 2
print(c1())  -- 3

local c2 = MakeCounter()
print(c2())  -- 1  ← c2 有自己独立的 count
```

`count` 在 `MakeCounter` 返回后本该销毁，但因为返回的函数还"捕获"着它，所以 Lua 会把它保留在闭包里。**每次调用 `MakeCounter()` 都会创建一个新的 `count`，所以 c1 和 c2 互不影响**。

### 游戏里闭包的典型用法

**用法1：事件回调 / UI 按钮绑定**

这是游戏开发中最常见的闭包场景：

```lua
-- 给背包格子绑定点击事件
for i = 1, 20 do
    local slotIndex = i  -- 注意：必须用 local 捕获！
    UI.GetSlot(i):SetOnClick(function()
        print("点击了格子 " .. slotIndex)
        BagSystem:UseItem(slotIndex)
    end)
end
```

⚠️ **经典陷阱**——循环里不加 `local` 会翻车：

```lua
-- ❌ 错误写法：所有按钮都打印 21！
for i = 1, 20 do
    UI.GetSlot(i):SetOnClick(function()
        print("点击了格子 " .. i)  -- i 是循环变量，结束后 i = 21
    end)
end

-- ✅ 正确写法：用 local 在每次迭代创建独立的 upvalue
for i = 1, 20 do
    local index = i  -- 每次循环创建新的 local → 新的 upvalue
    UI.GetSlot(i):SetOnClick(function()
        print("点击了格子 " .. index)
    end)
end
```

> 小知识：Lua 5.3+ 的 `for` 循环变量其实每次迭代都是新的 local（和 5.1 不同），但为了代码可读性和兼容性，养成手动 `local` 的习惯没毛病。

**用法2：模块私有状态（替代 static）**

```lua
-- Lua 没有 private/static，用闭包模拟
local function CreateSkillCDManager()
    local cdMap = {}  -- 私有数据，外部无法直接访问
    
    local M = {}
    
    function M.StartCD(skillId, duration)
        cdMap[skillId] = os.clock() + duration
    end
    
    function M.IsReady(skillId)
        local endTime = cdMap[skillId]
        return not endTime or os.clock() >= endTime
    end
    
    function M.GetRemaining(skillId)
        local endTime = cdMap[skillId]
        if not endTime then return 0 end
        return math.max(0, endTime - os.clock())
    end
    
    return M
end

local SkillCD = CreateSkillCDManager()
SkillCD.StartCD(1001, 5.0)
print(SkillCD.IsReady(1001))       -- false
print(SkillCD.GetRemaining(1001))  -- ~5.0
-- cdMap 完全不可从外部访问，天然防篡改
```

### 闭包的内存陷阱

闭包让 upvalue "活得更久"，但如果你不注意，就会造成**内存泄漏**。这是面试常考的坑：

```lua
-- ❌ 内存泄漏示例：闭包持有大对象的引用
function BindMonsterDeath(monster)
    -- monster 是个大对象（有模型、特效、AI数据……）
    EventSystem:On("MonsterDeath", function(id)
        if id == monster.id then
            PlayDeathAnim(monster)  -- 闭包捕获了整个 monster
        end
    end)
end

-- 问题：monster 被打死、从场景移除后，
-- 这个回调还持有 monster 引用 → monster 无法被 GC！
```

**修复方案：只捕获必要的值，及时注销回调**

```lua
function BindMonsterDeath(monster)
    local monsterId = monster.id  -- 只捕获 id，不捕获整个对象
    
    local handler
    handler = EventSystem:On("MonsterDeath", function(id)
        if id == monsterId then
            PlayDeathAnim(monsterId)
            EventSystem:Off("MonsterDeath", handler)  -- 用完就注销
            handler = nil
        end
    end)
end
```

一个简单的自查口诀：**"闭包捕获了啥？捕获的东西什么时候释放？回调什么时候注销？"**

## 💡 实战技巧

### 1. 游戏中常用的元方法速查

面试可能会问"你用过哪些元方法"，记住这几个就够了：

| 元方法 | 触发时机 | 游戏常见用途 |
|--------|---------|------------|
| `__index` | 读取不存在的 key | 类继承、配置表默认值、延迟加载 |
| `__newindex` | 给不存在的 key 赋值 | 数据变更监听（脏标记）、只读保护 |
| `__tostring` | `tostring()` 调用时 | Debug 打印对象信息 |
| `__gc` | 对象被 GC 回收时 | 释放 C 层资源（纹理、Socket） |
| `__len` | `#` 运算符 | 自定义容器长度 |
| `__eq/__lt/__le` | 比较运算 | 向量比较、版本号比较 |
| `__call` | 把表当函数调用 | 工厂模式 `MyClass()` 代替 `MyClass.New()` |

### 2. `__newindex` 实现数据变更监听

MVVM 绑定经常需要"数据一改，UI 自动刷"，`__newindex` 是 Lua 实现的核心：

```lua
-- 简易的可观察对象
function Observable(data, onChange)
    local proxy = {}
    local mt = {
        __index = data,
        __newindex = function(t, k, v)
            local old = data[k]
            if old ~= v then
                data[k] = v
                onChange(k, old, v)  -- 通知变更
            end
        end
    }
    return setmetatable(proxy, mt)
end

-- 使用：
local playerData = Observable({ hp = 100, mp = 50 }, function(k, old, new)
    print(string.format("[数据变更] %s: %s → %s", k, old, new))
    UIManager:Refresh(k)  -- 自动刷 UI
end)

playerData.hp = 80  -- [数据变更] hp: 100 → 80
playerData.mp = 50  -- 不触发（值没变）
```

### 3. 性能注意事项

| 场景 | 建议 |
|------|------|
| 热路径上频繁创建闭包 | 避免！每次创建闭包都有 GC 开销。战斗循环里不要写 `function() end` |
| `__index` 链太深（5层+） | 查找变慢。超过 3 层考虑把常用方法 copy 到子类表里 |
| upvalue 数量过多 | 一个闭包最多 255 个 upvalue（Lua 5.1 上限约 60 个）|
| 配置表用 `__index` 设默认值 | 合适！读取频率低，灵活性高 |

### 4. 面试中如何回答 "Lua 怎么实现面向对象"

说出这个结构，面试官就知道你真写过：

```
1. 表当类，元表连继承链
2. __index 实现方法查找（实例→子类→父类）
3. 构造函数里 setmetatable 关联实例和类
4. 冒号语法 : 是语法糖，自动传 self
5. 闭包做私有成员，外部完全不可访问
```
